# 好用工具包推荐

# 1. npm-run-all

一个用于串行或者并行执行 npm script 语句的工具，默认按顺序执行命令

开源仓库：https://github.com/mysticatea/npm-run-allopen in new window

参数：

-p: 并行执行 -s: 顺序执行

# 2. rimraf

一个用于快速删除文件或文件夹的包，类似于 rm -rf

开源仓库：https://github.com/isaacs/rimrafopen in new window

# 3. del

功能和 rimraf 类似

开源仓库：https://github.com/sindresorhus/delopen in new window

# 4. microbundle

一个用于很少配置即可打包js library 的包，基于 Rollup

开源仓库：https://github.com/developit/microbundleopen in new window

# 配置

{
  "name": "foo",                      // your package name
  "type": "module",
  "source": "src/foo.js",             // your source code
  "exports": {
    "require": "./dist/foo.cjs",      // used for require() in Node 12+
    "default": "./dist/foo.modern.js" // where to generate the modern bundle (see below)
  },
  "main": "./dist/foo.cjs",           // where to generate the CommonJS bundle
  "module": "./dist/foo.module.js",   // where to generate the ESM bundle
  "unpkg": "./dist/foo.umd.js",       // where to generate the UMD bundle (also aliased as "umd:main")
  "scripts": {
    "build": "microbundle",           // compiles "source" to "main"/"module"/"unpkg"
    "dev": "microbundle watch"        // re-build when source files change
  }
}

# 5. documentation

文档生成工具

# 6. jsdoc

文档生成工具

# 7. terser

一个 javascript es6+ 代码压缩工具

# 8. npm-check-updates

一个查找package.json 依赖升级到最新版本，忽略指定版本

# 9. vite 相关

# 9.1 vite-plugin-environment

将环境变量暴露到 import.meta.env，该方法和 webpack 的 EnvironmentPlugin 插件功能类似

# 9.2 vite-plugin-mock

vite 数据 mock 插件

# 9.3 vite-plugin-webpackchunkname

让 vite 兼容 webpack 的命名 chunkname 功能，支持下面的写法

import(/* webpackChunkName: "detail" */ "@/detail/somepage.vue")

# 9.4 vite-plugin-compression

文件压缩工具

# 10 picocolors

用于在终端输出带样式字符的包，功能和chalk类似，但是比chalk更小、更快

# 11 chalk

功能和上面的 picocolors 类似

# 12 prompts or enquirer or inquirer

这三者都是用来实现命令行交互式界面的工具

# 13 cac

一个用于构建 CLI 应用程序的 JavaScript 库，类似于 commander or yargs

# 14 semver

一个语义化版本号管理的包，帮助我们去判断一个版本是否合法，判断版本号命名是否正确，两个版本谁大谁小之类

# 15 minimist

一个命令行参数解析工具

// 使用
const args = require('minimist')(process.argv.slice(2))
// 例子
//  执行以下命令
vite create app -a 1 -b 2 -c5 -mn --bb=bb foo bar baz

// 将获得
{ _: [ 'foo', 'bar', 'baz' ],
  x: 3,
  y: 4,
  n: 5,
  a: true,
  b: true,
  c: true,
  beep: 'boop'
}

# 16 magic-string

是一个用于操作字符串和生成源映射的小而快的库，它最主要的功能就是对一些源代码和庞大的 AST 字符串做轻量级字符串的替换

import MagicString from 'magic-string';
const s = new MagicString('problems = 99');

// 替换 problems -> answer
s.overwrite(0, 8, 'answer')
s.toString() // 'answer = 99'

// 生成 sourcemap
var map = s.generateMap({
  source: 'source.js',
  file: 'converted.js.map',
  includeContent: true
})

# 17 fs-extra

一个强大的文件操作库， 是 Nodejs fs 模块 的增强版

# 18 chokidar

一款专门用于文件监控的库，解决 nodejs 官方api fs.watch 和 fs.watchFile 兼容不好和监听多次的性能问题

# 19 fast-glob

一个快速批量导入、读取文件的库，vite 工具中 import.meta.glob 方法（如下）就是基于这个库来实现

# 20 debug

一个模仿 Node.js 核心调试技术的小型 JavaScript 调试程序，在适用于 Node.js 和 Web 浏览器 都可使用

# 21 dotenv

是一个零依赖模块，可将 .env 文件 中的环境变量加载到 process.env 中

# 22 esbuild

esbuild 是一个基于 Go 语言开发的 JavaScript 打包工具，被 Vite 用于开发环境的依赖解析；

# 23 rollup

是一个 JavaScript 模块打包器，可以将小块代码编译成大块复杂的代码

# 24 ws

是一个简单易用、速度极快且经过全面测试的 WebSocket 客户端和 服务器 实现；完全可以是 Socket.io 的替代方案

# 25 esno

一个基于 esbuild 的 TS/ESNext 的 Node.js 运行

{
  "scripts": {
    "start": "esno index.ts"
  },
  "dependencies": {
    "esno": "*"
  }
}

# 26 tsup

一个轻小且无需配置的，由 esbuild 支持的打包工具

{
  "scripts": {
    "dev": "pnpm run build -- --watch --ignore-watch examples",
    "build": "tsup src/index.ts --dts --format cjs,esm"
  },
}

# 27 vitepress

在 vuepress 的基础上实现的静态网站生成器

# 28 vitest

一个由 vite 提供支持的快速单元测试框架
