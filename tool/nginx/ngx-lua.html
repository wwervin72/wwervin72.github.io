    • openresty 相关 指令
    • 说明

OpenResty® 是一个基于 Nginx 与 Lua 的高性能 Web 平台，其内部集成了大量精良的 Lua 库、第三方模块以及大多数的依赖项。用于方便地搭建能够处理超高并发、扩展性极高的动态 Web 应用、Web 服务和动态网关。

如果已经安装了 Nginx，又想使用 OpenResty 的功能，可以参考 《Nginx编译安装Lua》open in new window 一文安装lua-nginx-module模块即可。

或者直接使用 openresty 官方的 nginx 镜像.

docker pull openresty/openresty

lua-nginx-module 第三方模块文档地址open in new window

# 引入 lua 相关文件

# set search paths for pure Lua external libraries (';;' is the default path):
# 我这里只使用了 ngx 相关的内容，所以只引入了 ngx 下的文件
lua_package_path '/usr/local/openresty/lualib/ngx/?.lua;;';

# openresty 相关 指令

  1. content_by_lua: 返回的内容使用 lua 代码
  2. content_by_lua_file：读取lua文件里的 lua 代码
  3. content_by_lua_block: 和 content_by_lua 类似，区别是代码不写在字符串中，而是包裹在一对括号里
  4. access_by_lua: 请求处理短语，对每个请求执行lua代码，注意这里面的代码都是在标准模块 ngx_http_access_module后执行
  5. access_by_lua_block: 和 access_by_lua 类似，区别是代码不写在字符串中，而是包裹在一对括号里

举几个栗子：

# 这里直接使用官方提供的镜像
# 拉取镜像
docker pull openresty/openresty:latest
# 跑起来
docker run --name testopenresty -p 9999:80 openresty/openresty:latest &
# 进入到容器内部，尝试使用 openresty 带的方法修改 nginx.conf
docker exec -it be44561687b0 sh

content_by_lua_file 指令用法

# 创建 lua 文件
cd /usr/local/openresty/nginx/conf/
vim hello.lua

写入以下内容

<!--/usr/local/openresty/nginx/conf/hello.lua-->
ngx.say("<p>hello, lua</p>")

# 调整 ng 配置
location /hello {
    default_type text/html;
    content_by_lua_file /usr/local/openresty/nginx/conf/hello.lua;
}
# 重启 ng 
/usr/local/openresty/nginx/sbin/nginx -s reload
# 退出容器访问
curl http://localhost:9999;

# 说明

  1. ngx.req.get_uri_args() 用于获取URI请求参数。
  2. ngx.HTTP_BAD_REQUEST 为 ngx 常量，指的是400。代码里尽量使用常量。
  3. ngx.time() 用于获取时间戳。
  4. ngx.md5() 用于生成 md5 值。
  5. ngx.var.{变量名} 用于获取 nginx 里定义的变量，可以使用 ngx.var.[变量名] = $val 的形式修改外部变量。
  6. local aa = '123'; 定义 aa 变量为字符串123。
  7. require是 lua 里面引入其他库的关键字。
  8. 当我们要读取 http里的post数据的时候，就需要使用ngx.req.read_body()。该API同步读取客户端请求主体而不阻塞Nginx事件循环。
  9. ngx.req.get_post_args() 用于获取post请求参数。
  10. http 状态码变量查看，https://github.com/openresty/lua-nginx-module#http-status-constantsopen in new window
  11. ngx.ctx: 可以存储每一个请求的 lua 上下文数据，在当前请求结束之前可用。
  12. ngx.req.get_headers() 用于读取nginx的header参数。返回的是lua table
  13. ngx.decode_base64()用于 base64字符串解码。对应的编码API是 ngx.encode_base64()。

举个栗子. ngx.ctx 用法

location /test {
     rewrite_by_lua_block {
         ngx.ctx.foo = 76
     }
     access_by_lua_block {
         ngx.ctx.foo = ngx.ctx.foo + 3
     }
     content_by_lua_block {
         ngx.say(ngx.ctx.foo)
     }
}
