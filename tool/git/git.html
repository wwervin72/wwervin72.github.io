# 一些常用操作

# 提交本地所有修改内容到暂存区
git add -A

# 同上
git add .

# 将暂存区内容添加到本地仓库中
git commit -m 'some msg'

# git add -A 和 git commit -m 快捷操作
git commit -a -m''

# 将本地仓库的提交推送到远程仓库的 master 分支
 git push origin master

# 查看分支情况

# 查看本地分支
git branch
# 查看所有分支（包括远程分支）
git branch -a

# 远程仓库

# 本地仓库绑定远程仓库
git remote add origin url

# 查看本地添加的远程地址
git remote -v

# 删除远程地址
git remote remove origin

# 常见错误处理

# 1. fatal: refusing to merge unrelated histories

github 新建的创库与本地的仓库合并，两个仓库不同，所以出现这种错误, 需要这么写

git pull origin master --allow-unrelated-histories

# 2. fatal: The current branch master has no upstream branch.

git push -u origin master

# git clone 带上 账号密码

git clone -b ${BRANCH} https://${GIT_USERNAME}:${GIT_PASSWORD}@${GIT_URL}/${respository}.git

如果 GIT_USERNAME 中包含 @ 符号，需要将 GIT_USERNAME 中到 @ 修改为 %40

# git忽略跟踪已在远程仓库里的文件

  1. git rm --cached file (把文件从暂存区index中删除)
  2. 在 .gitignore 中添加此此文件
  3. git commit -m 'msg'
  4. git push

# 拉取远程分支

<!--拉取分支-->
git fetch origin branchName
<!--使用拉取得分支创建新分支-->
git checkout -b newBranchName origin/branchName

# 删除分支

# 删除本地分支
git branch -d ${branch}
# 删除远程分支
git push origin --delete ${branch}

# Tag 操作

# 获取远程tag
git fetch origin tag {{tagName}}

# 删除远程tag
# 删除本地
git tag -d ${tagName}
# 删除远程 tag
git push origin :refs/tags/${tagName}

# 恢复误删的本地分支

<!--已经推出 terminal 的，可以查看之前的 commit SHA1值-->
git reflog 
<!--没有退出的，在删除的时候会有一个sha值1，利用这个sha1值，去恢复分支-->
git branch -D master2
> Deleted branch master2 (was 130d7ba).    <-- This is the SHA1 we need to restore it!
git branch master2 130d7ba

# 撤销文件改动

# 撤销指定文件改动
git checkout -- {{filename}}
# 撤销所有文件改动
git checkout .

# commit 相关

# 切换到指定 commit
git checkout ${commitId}
# 查看 commit 提交的详细信息
git show ${commitId}

# 暂存改动 git stash

# 保存当前未commit的代码
git stash

# 保存当前未commit的代码并添加备注
git stash save "msg" 

# 列出stash的所有记录
git stash list

# 删除stash的所有记录
git stash clear

# 应用最近一次的stash
git stash apply

# 应用最近一次的stash，随后删除该记录
git stash pop

# 删除最近的一次stash
git stash drop

# 应用某条记录
git stash apply stash@{1}

# 回退版 reset

回退版本到某次提交，会丢弃掉目标提交到当前的提交，有个选项 --hard 或者 --soft，hard 是指直接丢弃目标提交到当前提交的内容，--soft 虽然会丢掉提交，但是这些提交改动的内容会变为 modified 状态

# 回退到指定的某次提交
git reset --soft {{commitID}}

# revert

给定一个或多个现有提交，恢复相关提交引入的更改，并记录一些这些更改的新提交。这就要求你的工作树是干净的（没有来自头部的修改），将现有的提交还原，恢复提交的内容，并生成一条还原记录。

# 撤销某次提交修改的内容
git revert commitID

# cherry-pick

# 合并某个 commit 到分支
git cherry-pick commitID

# 一次转移多个提交
git cherry-pick commitID1 commitID2

# 多个连续的commit，区间复制
git cherry-pick commitID1^..commitID2

# 如果有冲突，自动提交，则需要解决冲突后 git commit 手动进行提交，或者 git add . 后直接使用 git cherry-pick --continue 继续
git add .
git cherry-pick --continue

# 放弃 cherry-pick，回到操作前的样子，就像什么都没做过
gits cherry-pick --abort

# 中断操作，当前分支中未冲突的内容状态将为 modified
git cherry-pick --quit

# reflog

此命令管理重录中记录的信息，它记录了所有的 commit 操作记录，便于错误操作后找回记录。他和 git log 的区别是，如果我们使用了 git reset --hard commitID后，git log 则会找不到 commitID 之后的提交。但是 git reflog 是会记录所有的操作记录，可以使用它找到 commitID 之后的提交便于错误 reset 后恢复

git reflog
