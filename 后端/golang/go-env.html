# 运行go env 查看环境变量

# 1. GOARCH

CPU架构

# 2. BOGIN

工作目录下的bin文件夹

# 3. GOEXE

生成可执行文件的后缀

# 4. GOHOSTARCH GOHOSTOS

这两个环境变量在交叉编译的时候会用到。

  1. GOHOSTARCH想要交叉编译的CPU架构
  2. GOHOSTOS想要交叉编译的系统

# 4. GOOS

当前的操作系统名称

# 5. GOPATH

工作目录，该目录下有三个文件夹分别是bin、pkg、src

  1. bin用于存放编译后生成的可执行文件
  2. pkg用于存放编译后生成的包文件
  3. src存放项目的源码

# 5.1 配置GOPATH目录

# 5.1.1 windows下

打开系统环境变量，新建一个GOPATH的变量，输入工作目录的路径，如果有多个工作目录，则使用分号分隔

# 6. GOROOT

安装目录

# 7. GOTOOLDIR

工具目录
