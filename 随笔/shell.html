      • 1.1.1 #!/bin/bash
    • 1.1.2 #!/usr/bin/env node

# 1. 指定脚本解释器

支持两种写法：

1. 直接指定完整的解释器安装路径
2. #!/usr/bin/env，后面跟该环境中安装的环境变量，执行的时候会去$PATH中找对应的执行命令

#!/bin/bash
#!/usr/bin/python

#!/usr/bin/env bash
#!/usr/bin/env node

# 1.1 两种写法的优缺点

# 1.1.1 #!/bin/bash

优点：

1. 准确指出所需解释器的位置
2. 安全性相对较高
3. 可以传递多个参数

缺点：

1. 可移植性较差，不同的系统解释器位置可能不同
2. 一些解释器位置不太好记

# 1.1.2 #!/usr/bin/env node

优点：

1. 可移植性很强，系统自己去找所需命令
2. 不了解主机环境的时候，这种写法可以快速的开始开发工作

缺点：

1. 无法传递多个参数，#!/usr/bin/env node -w avc 这种写法会提示 "node -w avc 找不到这个命令的错误"
2. 安全性不够高，因为 $PATH中的命令可以被伪造，但是 /bin 目录一般只有root用户才有操作权限，所以 #!/bin/bash 写法相对更加安全
