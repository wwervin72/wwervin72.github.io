vue-router 中路由支持三种模式分别是：

  1. hash
  2. history
  3. abstract（nodejs 环境）

我们这里要说的是浏览器环境，也就是 hash、history。默认情况下 vue-router 使用 hash 模式

那么 hash 是什么呢，如果有这样一个 url：https://www.example.com/aaa?b=123#tag，我们可以在浏览器的控制台中打印出 location，可以发现下面的内容，hash 就是 # 后面的参数，所以我们在使用 hash mode 的时候，浏览器的地址栏中总是会类似于这样：http://localhost:8000/#/

# 1. hash 模式

我们发现在切换路由时，浏览器并不会去刷新界面，最外层使用的仍然是 index.html，这是因为 vue-router 内部使用了 history.pushState 的方式来往浏览器的历史记录堆栈顶部添加一条记录。我们可以通过 window.onhashchange 事件来监听到 hash 的变化，从而进行响应的操作。

<!--下面的代码就实现了，通过修改浏览器的 hash 值，来切换不同的 body 背景色，原理与 vue-router 类似-->
<!--同样我们所熟知到百度云盘也是才用的 hash 路由-->
// https://pan.baidu.com/disk/home?#/all?path=%2F&vmode=list
window.onhashchange = function(event){
    let hash = location.hash.slice(1);
    document.body.style.backgroundColor = hash;
}

# 2. history 模式

得益于 history api，我们可以去掉 url 中看起来比较丑陋的 # 了，通过 http://localhost:8000/ 这种方式我们也可以访问到我们的页面，不过要使用这种方式需要一些额外的配置。

# 2.1 使用 history mode

首先我们需要在路由文件中添加以下配置

new vueRouter({
    mode: 'history',
    routes: []
})
# 2.2 配置 nginx 和 404 路由

你以为这样就完了？ so naive!

完成了上面的配置后，运行 npm run build 将代码打包，然后将打包生成的文件放到 nginx 中，就可以进行访问了。通过切换路由，并且刷新浏览器，你会发现除了 / 跟路由其他的路由都是 404。

这是因为我们刷新浏览器时，会去请求服务器的当前页面所在路由的地址，实际上在 nginx 中我们只有一个 html 页面，并且其他的路由地址我们都没有配置，所以我们只能请求到 / 路由。而 hash 模式之所以不会出现这种情况，是因为他的路由都是放在 hash 中，从头到尾请求的服务器地址都是 /。这时候我们就需要给 nginx 添加一些设置，让他在任何路由下都给我们返回 index.html 页面的内容。

<!--nginx 配置-->
location / {
  try_files $uri $uri/ /index.html;
}

<!--并且需要给 vue-router 在增加一些额外的配置，匹配一些不存在的路由到指定的组件-->
export default new VueRouter({
  mode: 'history',
  routes: [
    { path: '*', component: NotFoundComponent }
  ]
})
# 2.3 history 原理

通过 history 的 go、back、forward，我们可以进行页面的前进。后退等操作。

history.go(-1);//后退一次
history.back(); //后退
hsitory.forward(); //前进

通过 pushState、replaceState，我们可以去修改浏览器历史栈中的数据。这两个方法接收 stateObj，title，url 三个参数，title 除了 safaris 其他的浏览器都忽略了该参数。而我们可以将某个特定路由的状态保存在 stateObj 中，比如滚动条的位置等。当页面的url再变回这个 url 时，可以通过 event.state 取到这个 state 对象，从而拿到我们存储的数据

history.pushState({pos: {x: 0, y: 100}}, 'pos', '/pos')
<!--当浏览器历史记录更改时，将触发popstate事件。-->
window.onpopstate = function(evt){
    console.log(evt.state)
}

调用 history.pushState() 或 history.replaceState() 去修改浏览器历史记录时，将不会触发 onpopstate 事件。只有用户手动进行浏览器的前进、后退、跳转或者执行 history.back()、 history.go()、history.forward() 等操作才会触发。
