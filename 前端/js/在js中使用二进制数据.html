对多数前端来说，在平常的工作中比较少在js中操作二进制数据，所以这也成了非常容易忽视的一个知识点，今天这里就总结一下这些内容。

js内置了一些全局变量，用于二进制数据以及其相关的处理，比如Blob, ArrayBuffer, Buffer（nodejs 提供）, File, FileReader, TypedArray, DataView，其中用的相对比较多的就是 Blob, ArrayBuffer, File, FileReader。使用场景通常集中在文件的上传，下载，文件转换等，下面来先来几本了解一下这些类型

  1. Blob 它是一个不可变的，原始数据的类文件对象，它的数据可以按文本或二进制的格式进行读取，也可以转换成 ReadableStreamopen in new window 来用于数据操作
  2. File 他继承于 Blob 对象，拥有 BLob对象的方法以及属性，提供文件相关的信息，他可以是网页中 input[type=file]标签选中文件后返回的 FileListopen in new window 对象，也可以是自由拖放操作生成的 DataTransferopen in new window对象，
  3. ArrayBuffer 用来表示通用的、固定长度的原始二进制数据缓冲区，也可以称它为字节数组。我们并不能直接操作他，而是要通过类型数组对象 (TypedArray) 或者是 DataView来操作
  4. Buffer 由 nodejs 提供，用于表示固定长度的字节序列
  5. FileReader 它允许Web应用程序异步读取存储在用户计算机上的文件（或原始数据缓冲区）的内容，使用 Fileopen in new window 或 Blobopen in new window 对象指定要读取的文件或数据。通常都是使用它来做数据类型的转换
  6. TypedArray 类型化数组，实际上在全局并没有一个 TypedArray 的全局变量，他实际上是一组类型数组的统称，主要包括：Int8Array, Int16Array, Int32Array, Uint8Array, Uint16Array, Uint32Array, Float32Array, Float64Array等
  7. DataView 是一个可以从二进制ArrayBufferopen in new window 对象中读写多种数值类型的底层接口

# Blob

上面我们说到 Blob 是一个类文件对象，我们通过 input[type=file]选择的文件也是继承自 Blob 的。所以我们使用它来进行文件上传和下载

# Blob 分段上传

async function selectFile(fileList) {
    const [file] = fileList
    const size = file.size
    const chunkSize = 30
    let start = 0
    try {
        while (start < size) {
            const form = new FormData()
            const end = start + Math.min(chunkSize, size - start)
            form.append('file', file.slice(start, end))
            const res = await axios.post('/xxx', form)
            start = end
        }
    } catch(_) {}
}

# Blob 转 Data url

/**
 * 将Blob转换为 Data Url
 * @param {*} streamData 
 * @returns Promise<DataUrl>
 */
export function convertBlobToDataurl(blob) {
    return new Promise((resolve) => {
        const reader = new FileReader()
        reader.readAsDataURL(blob)
        reader.onload = function () {
            resolve(reader.result)
        }
    })
}

# 二进制流数据转换为 Data url

/**
 * 将二进制流转换为 base64
 * @param {*} streamData 
 * @returns 
 */
export async function convertStreamDataToDataurl(streamData, type = 'application/pdf') {
  const blob = new Blob([streamData], {type})
  return await convertBlobToDataurl(blob)
}

# ArrayBuffer 的写能力

上面说到的 Blob主要是针对文件以及数据转换的操作，它是一个不可变的对象，如果我们要修改二进制数据的内容，那我们就需要用到 ArrayBuffer，下面通过一个 Int8Array 及 DataView 来举例

const typedArr1 = new Int8Array(8)
typedArr1[0] = 16
typedArr1[1] = 24
console.log(typedArr1)
// output: Int8Array(8) [16, 24, 0, 0, 0, 0, 0, 0, buffer: ArrayBuffer(8), byteLength: 8, byteOffset: 0, length: 8]

const buffer = new ArrayBuffer(16)
const view = new DataView(buffer)
view.setInt8(0, 16)
view.setInt8(1, 24)
console.log(view)
console.log(view.getInt8(0))
// output: DataView(16)
// output: 16

注意点：虽然 ArrayBuffer 看起来像个数组，但是他和数组还是有很大的区别，主要存在以下几点

  1. ArrayBuffer 是一个固定长度的二进制缓冲区，他的大小在初始化的时候就已经确定了，而数组可以随意的变换大小
  2. ArrayBuffer 缺少 Array 常见的一些 api，比如 push, pop, shift, unshift等
  3. ArrayBuffer 只能读，不能写，如果要修改数据内容，需要转换成 TypedArray 或者 DataView

# Buffer

Buffer 是 Nodejs 提供的对象，前端没有。它一般应用于 IO 操作，例如接收前端请求数据时候，可以通过 Buffer 的 API 对接收到的前端数据进行整合

exports.getUploadData = function (req, res, next) {
	const chunk = []
	req.on('data', bf => {
		chunk.push(bf)
	})
	req.on('end', bf => {
		const bf = Buffer.concat(chunk)
		console.log(bf.toString());
	})
}
